# backend-clean-arch — コア/ポート境界の強制（ヘキサゴナル最小形）

フル 4 層のクリーンアーキテクチャは強制しない。研究上の裏付けがあるのは「**境界を機械検査
できる**」ことの価値であって層の数ではない（AI ワーカーのコードベースで層分離が gate 通過率を
改善する定量研究は 2026-07 時点で存在しない）。このパックは最小形だけを強制する:

- **コア**（ドメイン語彙・ユースケース）は**内向きのみ**依存する。コアのディレクトリから
  インフラ/フレームワークへの import を禁止（L2: 編集時ブロック、L3: マージ済みツリー検査）。
- **ポート**（境界インターフェース）の IO 契約が受け入れ基準。過程は問わない
  （Spec Atlas 併用時は `design/oracle/` がその期待値表になる）。
- 単体テストはコア内で完結できる（依存が内向きなのでモック不要域が最大化される）。

## 適用後にやること
`gate.d/clean-arch.env` の `CORE_DIRS`（コアのパス接頭辞）と `FORBIDDEN_IMPORT_REGEX`
（コア内で禁止する import パターン）をプロジェクトに合わせて設定する。未設定の間、
ガード/チェックは何もブロックしない（助言のみ）。
