## フロントエンドのテスト方針（frontend-humble-object パック）
- **Humble Object**: 分岐・整形・状態遷移などのロジックは view（コンポーネント/テンプレート）
  から純粋関数・プレゼンタ・カスタムフックへ抽出し、そこへ単体テストを併設する。view は
  「謙虚」に薄く保ち、view 自体のテストは書かない。
- **E2E テストスイートを新設しない**: `e2e/`・`cypress/`・`*.e2e.*`・`*.cy.*` 等の作成は
  ハーネスがブロックし、gate も検査する。開発中にブラウザで動作確認するのは自由 — リポジトリに
  スイートとして固定しないこと。
- クリティカルパスの最小スモークは監督が `tests/` に置く（ワーカーの管轄外）。
- ロジックを view に埋めたままにするのは「テスト不能な実装」であり、レビューで差し戻される。
