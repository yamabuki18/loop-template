# frontend-humble-object — E2E 縮退 + Humble Object

「E2E は TDD に向かずリグレッション検知が主役割になる」を方針として採用しつつ、研究知見
（E2E/ブラウザ検証ツールはエージェントの自己検証手段として性能を劇的に改善する）との衝突を
避けるため、**全廃ではなく役割限定**にする:

- **恒常的な E2E リグレッションスイートを作らない**: `e2e/`・`cypress/`・`*.e2e.*`・`*.cy.*`
  等の新設を L2（編集時）と L3（マージ時）でブロック。
- **Humble Object**: ロジックは view から分離して純粋関数/プレゼンタに置き、単体テストで
  固定する。view は「謙虚」に薄く保ち、テストしない。
- **開発時のブラウザ自己検証は禁止しない**: ワーカーが動作確認に使うのは自由（スイートとして
  リポジトリに固定することだけを禁じる）。
- **最小スモーク**はクリティカルパスに限り監督（計画役）が `tests/` に置く — ワーカーは
  そもそも `tests/` を触れないので、この配分は構造的に守られる。

## 適用後にやること
`gate.d/frontend-testing.env` の `E2E_BLOCK_REGEX` をプロジェクトの慣習に合わせて調整する
（既定で e2e/cypress/playwright の一般的パターンをブロックする）。
