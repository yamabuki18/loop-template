# ARCHITECTURE — 構造とディレクトリの地図

> 計画役がスライスを「縦割り（各ワーカーが自分のディレクトリだけ触る）」に分解するための地図。
> ここが正確だと owned-paths の衝突が減り、ワーカーが毎回コードを探索し直さなくて済む。

## スタック
- 言語 / ランタイム: <例: TypeScript / Node 22>
- フレームワーク: <例: Next.js, Express, FastAPI ...>
- テスト: <例: vitest / pytest / go test>　← gate が走らせるチェックと一致させる

| パス | 責務 | 想定オーナー |
|------|------|--------------|
| `src/<featureA>/` | <...> | 1ワーカー専有 |

- `package.json` / ロックファイル
